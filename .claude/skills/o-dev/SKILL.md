# 実装サイクル（オーケストレーション）

`/impl` → `/review` を連続で実行する。

このスキル自体はコードを書かず、ドキュメントも編集しない。単一責任スキルの実行順序を制御するだけ。

## Procedure

### 1. /impl を実行

`$ARGUMENTS` が指定されている場合はそのまま `/impl` に渡す。

```
/impl $ARGUMENTS
```

`/impl` の完了を待つ。途中でブロッカーが発生した場合は、`/impl` の報告を表示してこのスキルも停止する。

### 2. /review を実行

`/impl` が完了したら、コードレビューを実行する。

```
/review
```

### 3. 完了報告

```text
実装サイクル完了
═══════════════════════════════════════
実行したスキル:
  1. /impl — <タスク番号> <中タスク名>
  2. /review — <レビュー結果サマリー>

次のステップ:
  - 変更内容を確認して /commit でコミットする
  - 問題があれば修正して再度 /o-dev を実行する
═══════════════════════════════════════
```
