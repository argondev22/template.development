# フィードバック対応（オーケストレーション）

LOG.md の未対応フィードバックを分析し、必要なドキュメント更新とタスク化、実装、レビューまでを連続で実行する。

このスキル自体はコードを書かず、ドキュメントも編集しない。単一責任スキルの実行順序を制御するだけ。

## Procedure

### 1. /log を実行（分析）

```text
/log
```

AI コメントが付与された結果を確認する。未返信のつぶやきがない場合は停止する。

### 2. 対応方針をユーザーに確認

AI コメントの分析結果を提示し、対応するかどうかをユーザーに確認する。

対応しない場合はここで停止する。

### 3. 影響範囲に応じたドキュメント更新

AI コメントで推奨されたスキルに従い、上流から順にドキュメントを更新する:

**仕様変更が必要な場合:**

```text
/req → /design → /task <追加するタスクの内容>
```

**設計変更が必要な場合:**

```text
/design → /task <追加するタスクの内容>
```

**タスク追加のみの場合:**

```text
/task <追加するタスクの内容>
```

各スキルの実行後、ユーザーに確認を取ってから次に進む。

### 4. /impl を実行

追加されたタスクを実装する。

```text
/impl
```

### 5. /review を実行

```text
/review
```

### 6. 完了報告

```text
フィードバック対応完了
═══════════════════════════════════════
対応したフィードバック:
  - <LOG.md のつぶやき内容>

実行したスキル:
  1. /log — AI コメント付与
  2. </req, /design, /task のうち実行したもの>
  3. /impl — <タスク番号> <タスク名>
  4. /review — <レビュー結果サマリー>

次のステップ:
  - 変更内容を確認して /commit でコミットする
═══════════════════════════════════════
```
